# 网络与连接

## 连接超时

遇到连接超时、请求失败时，先用控制台里已经生成好的 API 请求做验证。

![在控制台复制 API 请求](/images/screenshots/console-copy-api-request.png)

操作方法：

- 登录 CCWSPro 控制台。
- 进入「概览」页面。
- 找到右侧「首个 API 请求」卡片。
- 点击右上角「复制」。
- 把复制出来的请求粘贴到终端或 PowerShell 里执行。

::: warning 注意
复制出来的请求里会包含你的 API Key，不要发到群聊、截图、公开仓库或文章正文里。
:::

如果复制出来的请求能正常返回内容，说明平台接口、Key 和模型调用基本可用。后续再检查 Codex、CC Switch 或其他工具里的配置是否填写一致。

::: danger 不要这样测
不要用 HEAD 请求去测 `/v1` 地址是否可用。`/v1` 是 API 前缀，不是普通网页地址；HEAD 请求不能代表真实模型调用是否可用。
:::

## 域名解析检查

如果控制台复制的 API 请求完全连不上，可以再检查基础域名是否能解析：

```bash
ping www.ccws.pro
```

`ping` 只能辅助判断域名解析和本机网络状态，不能替代真实 API 请求验证。

## DNS 问题

如果浏览器能打开官网，但命令行无法访问接口，可能是 DNS 或代理配置不一致。

建议：

- 切换网络环境。
- 关闭或调整代理。
- 检查系统 DNS。
- 在终端里单独验证 `www.ccws.pro`。

## 代理冲突

如果使用了代理工具，请确认 HTTP_PROXY、HTTPS_PROXY、ALL_PROXY 等环境变量没有误伤本地工具。
